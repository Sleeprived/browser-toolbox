    Cron syntax reference & tips

    A cron expression has five space-separated fields. Each field accepts the same operators:

    FieldAllowedNames
    Minute0–59—
    Hour0–23—
    Day of month1–31—
    Month1–12jan–dec
    Day of week0–7sun–sat (0 & 7 = Sunday)
    • * — every value
    • , — a list, e.g. 1,15,30
    • - — a range, e.g. 1-5 (Mon–Fri)
    • / — a step, e.g. */15 (every 15) or 9-17/2
    • Nicknames: @yearly @monthly @weekly @daily @hourly

    Good to know. When you set both day-of-month and day-of-week, cron runs when either matches (not both). Cron's finest resolution is one minute, and it does not back-fill runs missed while a machine is asleep.

    Note: extended syntax from other schedulers — L (last day), W (nearest weekday), # (nth weekday), and a leading seconds field — is not part of standard cron and isn't parsed here.
